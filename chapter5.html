4.10 System Block Diagram

The system block diagrams below describe the operational flow of Kinetix across four scenarios: Boot-Up, Passive Mode, Active Mode, and Emergency Override.

Boot-Up Sequence

When KINETIX is plugged in, the mechanical switch determines the device state. In the 'On' state, the device enters Warm-Up Mode — the MCU commands the motor driver (horizontal), which drives the motor and ball-screw nut, placing the footrest at the starting position (full flexion). The device then awaits commands from the tablet via Wi-Fi.

Figure 37
KINETIX Boot-Up Sequence flowchart — KINETIX plugged in → Mechanical Switch (Open → Off State / Closed → On State) → On State → Warm Up Mode → MCU → Motor Driver (Horizontal) → Motor → Ball-Screw Nut → Footrest at start position (Full Flexion) → Awaits tablet command to determine Active or Passive Mode.

Figure 37. KINETIX Boot-Up Sequence.

Passive Mode

In passive mode, the device automatically drives the footrest along the screw bar for horizontal motion and along the arc rail for lateral motion. An ultrasonic sensor is used to track the footrest's position during horizontal movement.

Figure 38
KINETIX Passive Mode Flowchart — Exercise commands issued; splits into Horizontal Movement path (MCU → Motor Driver Horizontal → Motor → Ball-Screw Nut → Linear Motion ← Ultrasonic Sensor feedback) and Lateral Movement path (MCU → Motor Driver Lateral → Motor → Arc-Rail Joint → Lateral Motion). Both paths loop.

Figure 38. KINETIX Passive Mode Flowchart — automatic footrest actuation for horizontal and lateral movement.

Active Mode

Active mode adds force and IMU sensors. The force sensors set a resistance threshold the user must overcome, recording exerted force to adjust difficulty and track progress. The IMU monitors range of motion, providing further rehabilitation performance data. Sensor data is transmitted via Wi-Fi to a database.

Figure 39
KINETIX Active Mode Flowchart — extends Passive Mode with: Ultrasonic + IMU + Force Sensors on horizontal path; IMU & Force Sensors on lateral path; Database node connected via Wi-Fi from MCU for data storage and progress tracking.

Figure 39. KINETIX Active Mode Flowchart — force and IMU sensing added for interactive rehabilitation.

Emergency Override

Upon pressing the physical emergency stop button (or issuing an emergency stop command from the tablet), the MCU immediately halts both motor drivers (horizontal and lateral), causing the device to cease all movement. The physiotherapist then uses the tablet to recalibrate the footrest to its starting position before beginning a new session.

Figures 40–41
KINETIX Emergency Override Sub-System (2 parts): Part 1 — Emergency Stop Button or Tablet Wi-Fi command → MCU → Motor Driver (Horizontal + Lateral) → Motors stop → KINETIX ceases movement. Part 2 — Tablet → Wi-Fi → Warm Up Mode → MCU → Motor w/ Encoder → Ball-Screw Nut → Footrest at start position (Full Flexion), completing reset cycle.

Figures 40–41. KINETIX Emergency Override system — immediate stop and reset via tablet recalibration.

Key System Properties
  • Two motors: one for horizontal (linear) movement, one for lateral movement.
  • Wireless control via Wi-Fi (Raspberry Pi Pico W2 MCU).
  • Ultrasonic sensor for horizontal position tracking in passive mode.
  • IMU + force sensors added in active mode for biomechanical feedback.
  • Physical emergency stop button plus tablet-based software stop.
